1
PC arma el paquete
La aplicación genera datos, TCP/UDP arma el segmento y la capa IP construye la cabecera con las IPs y el TTL inicial.
SRC: 192.168.1.10
DST: 203.0.113.10
TTL: 64 PROTO: TCP(6)
SIZE: 1500 bytes
→ Enviando al gateway 192.168.1.1
2
Router A: tabla de ruteo
El router recibe el paquete, decrementa el TTL y busca 203.0.113.0/24 en su tabla. No está en la red local → lo reenvía al siguiente salto hacia internet.
TTL: 64 → 63 (−1 por salto)
LOOKUP: 203.0.113.0/24
RESULT: via 0.0.0.0 → default route
→ Reenviando hacia ISP
3
Tránsito por internet
Múltiples routers intermedios (ISPs, IXPs) leen solo la IP destino y encaminan el paquete. Cada salto reduce el TTL en 1.
Salto 1: TTL 63→62
Salto 2: TTL 62→61
Salto 3: TTL 61→60
... (aprox. 10–15 saltos)
TTL actual: ~50
4
Router B: red destino
Router B reconoce que 203.0.113.10 pertenece a su red local (203.0.113.0/24). Usa ARP para obtener la MAC del servidor y entrega el paquete.
TTL: ~50 → ~49
LOOKUP: 203.0.113.10
RESULT: local → red 203.0.113.0/24
ARP: ¿quién tiene 203.0.113.10?
→ Entregando localmente
5
Servidor recibe el paquete
El servidor verifica que la IP destino coincide con la suya, acepta el paquete, extrae los datos y los pasa a la aplicación (ej. servidor web).
DST 203.0.113.10 == yo → ACEPTADO
PROTO: TCP(6) → puerto 80
TTL final: ~49
